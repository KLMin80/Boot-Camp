# COST.md — 운영비 산출 (OCR·레시피 LLM)

> 실측 토큰 × 확인된 단가로 계산. 추정이 아니라 **실제 호출 결과** 기반.
> gpt-4o-mini 단가(2026-07): 입력 $0.15/1M · 출력 $0.60/1M. 환율 ₩1,380/$.

## 1. 단위 비용 (실측)

### OCR (라벨 사진 1장, gpt-4o-mini 비전, detail:high)
- 실측 입력 토큰: 8,600 ~ 37,000 (사진 해상도 비례), **평균 ≈ 22,000** + 출력 ~60
- 계산: 22,000 × $0.15/1M + 60 × $0.60/1M ≈ **$0.0033 = ₩4.6 / 장**

⚠️ **다운스케일은 효과 없음 (실측 확인).** 3024px→1024px로 줄여도 토큰 그대로(OpenAI가 내부적으로 768px로 이미 축소) + 작은 라벨 정답이 틀어짐(2020.09.12→2023). **원본 그대로 detail:high가 최선.**

### 레시피 생성 (gpt-4o-mini 텍스트, **캐시 미스 때만**)
- 입력 ~620 + 출력 ~1,100 ≈ **$0.00075 = ₩1 / 생성**
- 하이브리드 캐시라 대부분 히트(₩0). 전역 카탈로그가 커질수록 미스가 0에 수렴 → **전체 비용의 1% 미만**

## 2. 가구당 월 비용

**가정(명시):** 장보기 8회/월, 그중 라벨 있는 포장식품 스캔 ~50개/월(신선식품은 아이콘 경로=무료). 레시피 요청 30회/월(초기 미스 ~3회).

| 항목 | 월 사용량 | 비용 |
|------|-----------|------|
| OCR 스캔 | 50장 | ₩230 |
| 레시피 생성(미스) | 3회 | ₩3 |
| **합계** | | **≈ ₩233 / 가구·월 ($0.17)** |

## 3. 규모별 월 비용

| 활성 가구 | OCR+레시피 월비용 | 연간 |
|-----------|-------------------|------|
| 1,000 | ₩23만 ($167) | ₩276만 |
| 10,000 | ₩233만 ($1,690) | ₩2,800만 |
| 100,000 | ₩2,330만 ($16,900) | ₩2.8억 |

> 레시피는 공유 캐시라 규모가 커져도 거의 안 늘고, **OCR이 비용의 95%+**.

## 4. 손익분기 — 쿠팡파트너스가 OCR비를 덮는가?

가구당 원가 ₩233/월. 쿠팡파트너스 식료품 수수료 ~2% 가정.

- **손익분기 주문액 = ₩233 / 0.02 ≈ 가구당 월 ₩11,650 주문**
- 즉 한 가구가 앱에서 **부족 재료를 월 1.2만원어치**만 사면 OCR비가 상쇄됨(월 2~3개 품목 수준)
- 그 이상 주문 시 흑자. **비용이 아니라 "쿠팡 전환율"이 관건.**

⚠️ **전환이 안 되면** ₩233/가구는 순수 지출. 10,000가구면 상쇄 없이 월 $1,690.

## 5. 비용 레버 (실측 기준, 효과 순)

| 레버 | 효과 | 상태 |
|------|------|------|
| **신선식품 아이콘 경로** (OCR 안 씀) | OCR 물량 자체를 줄임 | ✅ 이미 설계됨 |
| **무료 한도** (예: 월 30스캔 무료, 그 이상 프리미엄) | 헤비유저 비용 상한 | v1에 넣을 수 있음 |
| **detail:low + 실패 시 high 폴백** | 큰 라벨은 8배 저렴(₩0.5), 작은 건 자동 승급 | 복잡도↑·정확도 위험, 선택적 |
| 클라이언트 다운스케일 (기각) | ❌ 실측 0~31% + 정확도 저하 | 버림 |
| **온디바이스 OCR** (Capacitor Vision/ML Kit) | **₩0 영구** — 비용 자체가 사라짐 | Phase 5 엔드게임 |

## 6. 결론

- **개인/소규모(수백 가구)에선 비용이 사실상 무의미** — 월 몇만원.
- **1만 가구 규모**부터 월 $1,700가 실제 지출 → 이때는 **쿠팡 전환율**과 **무료 한도**가 흑자를 좌우.
- **엔드게임은 온디바이스 OCR**(Phase 5). 앱으로 감싸면 OCR비가 영구 ₩0이 되어 비용 리스크가 통째로 사라짐.
- **판단:** v1(웹, 클라우드 OCR)로 검증하기엔 비용이 충분히 낮다. 규모가 커지기 전에 온디바이스로 전환하면 됨. **진행 가능.**
