## 1차 요청 — 더치페이 계산기 만들기

**사용자**
> 더치페이 계산기 만들어줘. 인원수, 각자 먹은 메뉴, 할인 적용을 적용할 수 있도록 해줘. (배열, 반복문, 조건 분기 기능을 활용)
> `week-2\quest\더치페이 계산기` 폴더에 파일로 만들어줘

**작업 내용**
- `single-react-dev` 에이전트로 단일 `index.html` 생성.
- 구현 기능:
  1. **인원 관리** — 참여자 배열로 추가/삭제/이름 변경 (`map`, `filter`)
  2. **각자 먹은 메뉴** — 참여자마다 메뉴 배열을 갖는 **중첩 배열** 구조 (메뉴명 + 가격)
  3. **할인 적용** — 정률(%)/정액(원) 토글(조건 분기), 먹은 금액 비율대로 분배 차감
  4. **1/N 균등 분배 모드** — 총액을 인원수로 나누되 마지막 사람이 나머지 부담(1원 오차 방지)
  5. **정산 결과 표** — 참여자별 먹은 총액/할인/실제 낼 금액 + 요약, 엣지 케이스 안내
- 금액: `₩` 천단위 콤마, 한국어 UI, 반응형.

## 3차 요청 — 메뉴별 할인 + 공통 적용 버튼

**사용자**
> 각자 먹은 메뉴를 추가할 수 있도록 해주고, 할인적용도 각 메뉴별로 적용할 수 있도록 해줘. (할인율이 동일할 경우에는 공통적용 버튼도 만들어줘)

**작업 내용**
- 전체 금액 단일 할인 → **각 메뉴 항목별 개별 할인**으로 변경.
  - 메뉴 데이터에 `discountType`(정률/정액) + `discountValue` 필드 추가.
  - 새 함수 `calcMenuDiscount(menu)`(조건 분기), `menuPayable(menu)` 추가.
  - 메뉴 카드에 **원가 / 할인 / 적용가** 함께 표시 + 메뉴별 할인 입력 UI.
  - 참여자 부담액 = 메뉴들의 할인 적용가 합계(`menus.reduce`).
- **공통 적용 / 초기화 버튼** 추가.
  - "전체 메뉴에 공통 적용" → `participants.map → menus.map` 중첩 반복으로 일괄 반영.
  - "할인 초기화" → 전체 메뉴 할인값 0으로 리셋.
- **1/N 모드**는 기존 총액 단일 할인 방식 유지(메뉴별 할인 UI는 숨김).
- 검산 예: A(라면 8,000 정률10%, 김밥 3,000 정액500) + B(돈까스 9,000) → 합계 18,700 ✔️

## 핵심 학습 포인트 (배열 · 반복문 · 조건 분기)

| 기능 | 사용 문법 |
|------|-----------|
| 참여자/메뉴 추가·삭제 | `[...prev, new]`, `Array.filter` |
| 이름·할인 변경 | `Array.map` (불변 업데이트) |
| 메뉴 합계 계산 | `Array.reduce` 누적 |
| 1/N 나머지 분배 | `for` 반복문 + `Math.floor`, 마지막 인덱스 보정 |
| 할인 방식 분기 | `if (discountType === 'percent') ... else ...` |
| 전체 메뉴 일괄 적용 | `participants.map → menus.map` 중첩 반복 |
| 엣지 케이스 안내 | 조건 분기 + empty state 렌더링 |
