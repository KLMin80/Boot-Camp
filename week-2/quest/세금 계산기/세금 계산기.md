# 한국 세금 계산기 — 개발 기록

> Week 2 Quest · 단일 파일(`index.html`) React 앱

CDN 기반 React 18 + Tailwind CSS로 빌드 도구 없이 단일 `index.html`에 구현한 한국 세금 계산기입니다. 아래는 개발 과정에서 오간 요구사항과 구현 내용을 정리한 기록입니다.

## 1차 요청 — 세금 계산기 생성

> **요청 내용**
> 세금 계산기 만들어줘. 원하는 세금 종류(예: 종합소득세, 부가가치세 등)용 계산기로 세금 종류를 선택할 수 있게 해주고, 원하는 모든 세금을 중복 선택 가능하도록 구성해줘.

### 구현 내용
- **CDN 기반 React 18 + Tailwind CSS** 단일 파일(`index.html`) 구조
- **세금 종류 다중/중복 선택**: 체크박스로 원하는 세금을 여러 개 동시에 선택 → 선택한 만큼 계산기 카드가 동시에 표시되고, 해제하면 사라짐
- **실시간 계산**: 입력값 변경 즉시 결과 갱신, 천 단위 콤마 원화 표기
- **세율 데이터 상수 분리**: 2024~2025년 기준 세율표를 코드 상단에 상수로 정의, **누진공제 방식**으로 정확히 계산
- 하단에 **참고용 추정치 안내(Disclaimer)** 문구 포함

### 초기 지원 세금 (7종)
| 세금 | 계산 방식 |
|---|---|
| 종합소득세 | 과세표준 구간별 누진세율 + 누진공제, 지방소득세 10% |
| 부가가치세 | 공급가액 10%, 공급가액 ↔ 공급대가 변환 |
| 양도소득세 | 양도차익 − 기본공제(250만원) → 기본세율 누진 |
| 근로소득세(간이) | 연봉 → 근로소득공제 → 간이 추정 |
| 상속·증여세 | 구간별 누진세율 + 누진공제 (공제 단순 가정) |
| 재산세(주택) | 공시가격 × 공정시장가액비율 60% → 누진세율 |
| 자동차세(승용) | 배기량 cc당 세액 + 지방교육세 30% |

## 2차 요청 — 누락 세금 추가 + 합계 요약표

> **요청 내용**
> 혹시 근로소득자가 고려해야 할 만한 세금 중 누락된 것이 있다면 추가해줘. 추가한 이후에 세금 총 합계가 보여지는 요약집계표도 포함해줘.

### 추가한 항목 (근로소득자 관점)
| 추가 항목 | 내용 | 추가 이유 |
|---|---|---|
| **4대보험 (근로자분)** | 국민연금 4.5%, 건강보험 3.545%, 장기요양(건보료×12.95%), 고용보험 0.9% — 월·연 환산 | 매달 급여에서 빠지는 의무 공제(준조세). 실수령액에 가장 큰 영향. 국민연금 기준소득월액 상한(617만원) 반영 |
| **금융소득세 (이자·배당)** | 원천징수 15.4% (소득세 14% + 지방세 1.4%) | 예적금·배당 보유 직장인 다수 해당. 연 2천만원 초과 시 **금융소득종합과세 대상** 경고 표시 |

> 종합부동산세 등도 검토했으나, 일반 근로소득자 다수에 해당하는 위 두 항목을 우선 반영.

### 세금 합계 요약집계표
- 선택한 각 세금의 **부담세액**을 행으로 나열 + **총 합계** 강조 표시
- 항목별 **비중(%)** 과 색상 **비중 막대 그래프**
- 각 계산기가 산출 총액을 App으로 보고(`onResult` 콜백 → `results` state)하는 구조로, 입력값 변경 시 **합계 실시간 갱신**
- 항목별 과세 기준(연세액·1회성·원천징수)이 달라 단순 합산은 참고용이라는 안내 문구 포함
- 기본 선택을 근로소득자 중심으로 **근로소득세**가 켜진 상태로 시작하도록 변경

### 최종 지원 세금 (9종)

## 기술 구조 요약

```
index.html (단일 파일)
├─ <head>
│   ├─ Tailwind CSS (CDN)
│   ├─ React 18 + ReactDOM (CDN)
│   ├─ Babel standalone (JSX 변환)
│   └─ <style> 카드 진입 애니메이션 등 최소 커스텀 CSS
└─ <script type="text/babel">
    ├─ 세율 데이터 상수 (구간별 누진세율 + 누진공제)
    ├─ 유틸 함수 (won 원화 포맷, calcProgressive 누진계산 등)
    ├─ Design System 컴포넌트 (Card, Field, ResultRow, Badge, CalcCard)
    ├─ 개별 세금 계산기 컴포넌트 9종 (각자 onResult로 총액 보고)
    ├─ TaxSelector (다중 선택 체크박스)
    ├─ SummaryTable (합계 요약집계표)
    └─ App (선택 상태·결과 집계 관리)
```

### 핵심 설계 포인트
- **누진공제 방식**: `과세표준 × 세율 − 누진공제`로 구간별 세액 정확 계산 (`calcProgressive` 함수)
- **상태 끌어올리기**: 각 계산기가 내부에서 계산한 총 부담세액을 `onResult(amount)` 콜백으로 App에 보고 → App의 `results` state에 모아 요약표에서 합산
- **Tailwind CDN(JIT) 제약 대응**: 런타임 동적 클래스는 인식 못 하므로 색상 클래스(`COLOR_RING`, `COLOR_BG`, `COLOR_BAR`)를 정적 문자열로 매핑
- **반응형**: 모바일/태블릿/데스크톱 대응 (`grid-cols`, `sm:`/`md:`/`lg:` 프리픽스)
