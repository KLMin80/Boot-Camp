## 2. 작업 내용

`single-react-dev` 에이전트를 사용해 `C:\Boot Camp\week-2\07-goblin\index.html`에 단일 파일 React 웹앱을 생성했습니다.

### 기술 스택
- CDN 기반 React 18 (빌드 도구/번들러 없음)
- Tailwind CSS (CDN)
- Babel (JSX 변환)
- 외부 API 호출 없음 — 모든 색상 연산은 JS로 처리

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 색상 입력 양방향 동기화 | HEX 텍스트 입력과 네이티브 컬러 피커(`<input type="color">`)가 서로 연동 |
| HEX 유효성 검사 | 3자리/6자리, `#` 유무 모두 허용 (`#fff`, `3498db` 등). 잘못된 값은 입력창이 빨갛게 표시 |
| 팔레트 생성 | RGB↔HSL 변환으로 명도(Lightness)를 조절해 **밝게 5단계 + 기준색 + 어둡게 5단계 = 총 11개** 스와치 생성 |
| 클릭 복사 | 스와치 클릭 시 HEX가 클립보드에 복사되고 "복사됨!" 토스트 표시 |
| 가독성 자동 처리 | 각 스와치의 휘도를 계산해 검정/흰색 글자를 자동 선택, 기준색은 가운데에서 링으로 강조 |
| 추가 요소 | 추천 색상 프리셋 6개, 반응형 레이아웃 (모바일 2열 → 데스크톱 11열) |

### 구현 디테일
- `normalizeHex()` — 3자리/6자리, `#` 유무를 모두 허용하는 HEX 정규화 함수
- 직접 구현한 RGB↔HSL 변환으로 HSL 명도(Lightness)를 균등 분할
  - 밝은 쪽: 흰색(L=100) 방향
  - 어두운 쪽: 검정(L=0) 방향
- `navigator.clipboard` 사용 (구형 브라우저용 폴백 포함)
- 상대 휘도 계산으로 배경 대비 글자색 자동 선택
- aria 라벨, 시맨틱 마크업, 반응형 그리드 적용

## 3. 사용 방법

브라우저로 바로 열어도 되지만, 클립보드/컬러 피커가 안정적으로 동작하려면 로컬 서버 사용 권장:

```powershell
cd "C:\Boot Camp\week-2\07-goblin"
npx serve .
```

또는 VS Code의 Live Server 확장으로 열기.

### 사용 흐름
1. 컬러 피커나 HEX 입력으로 기준색 지정
2. 11단계 팔레트가 자동 생성됨
3. 원하는 스와치를 클릭하면 HEX가 클립보드에 복사됨
