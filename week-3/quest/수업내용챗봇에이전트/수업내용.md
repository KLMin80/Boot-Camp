## 2. Skill & Agent

### Skill
- `/` 입력 → 저장된 Skill 목록 표시
- Skill 저장 위치 2곳: ① 내 컴퓨터 루트, ② 해당 프로젝트 폴더
- `/skills` : 보유 Skill 확인
- 가장 직관적인 활용법: 이전 프로젝트의 `.claude` 폴더(skills/agents 포함)를 복사해서 새 프로젝트 폴더에 붙여넣기
- 호출 기호: **Skill = `/`**

### Agent
- 호출 기호: **Agent = `@`**
- **메모리(context 저장)가 있음** → 지속적으로 업데이트된 정보를 활용하는 작업에 적합

### Skill vs Agent 차이
| 구분 | Skill | Agent |
|------|-------|-------|
| 성격 | 단순 작업(매크로 한 번 실행 수준) | 인사이트를 가진 복합 작업 위임 |
| 예시 | 간단한 변환·반복 | "마케팅 플랜 짜줘" (여러 단계·여러 Skill 필요) |
| 메모리 | 없음 | 있음 (context 저장) |

> **Tip – Plugin**: plugin을 통해 Skill 묶음을 한 번에 설치 가능
> **Tip – Scheduled task** (Claude Cowork): 능동적으로 사용자에게 물어 관리. 예) Slack 연동해 매일 약 복용 여부 확인

### 새 Skill/Agent/프로그램 인식시키기
- `/reload` 실행 **또는** Claude를 껐다 다시 열기
- Agent 파일이 프로젝트 폴더 밖에 있으면 폴더 안으로 복붙 후 reload

## 5. 크롬 개발자도구로 API 확인

- 열기: 우측 상단 점 3개 → 도구 더보기 → 개발자도구 (또는 우클릭 → 검사)
- 주요 3개 탭:

| 탭 | 역할 |
|------|------|
| **Elements** | HTML 요소 확인 (브라우저에서 직접 HTML 구조 보기) |
| **Console** | JavaScript 영역(동적 부분). 예: `alert("안녕")` 입력 시 팝업. JS 에러 표시 |
| **Network** | JS가 호출한 API 확인. 항목 클릭 → Header에서 **Request URL** 확인 가능 |

- API 키 확인: Network 탭 → 항목 Header → Request URL에 `...&appid=API키정보` 형태로 노출됨

## 6. 백엔드 서버 개념 & Node.js

### 핵심 개념
- 웹서버를 쓴다 = **별도의 웹서버 레이어를 통해** 파일을 본다는 의미
- 개발자도구 Network → Header의 Request URL로 구분:
  - `file:///...` → 그냥 파일을 직접 읽는 것 (index.html을 브라우저로 바로 열 때)
  - `http://...` → 웹 프로토콜(웹서버 경유)
- 웹서버는 Java, Python, PHP, **Node** 등 다양한 언어로 제작 가능

### Node 서버 실행/종료
- 실행: 해당 폴더에서 (Claude 호출 없이) 터미널만 열고 `node 폴더명/server.js`
  - 성공 시: `서버가 http://localhost:3000 에서 실행 중입니다.`
- `node -v` : Node 설치 여부/버전 확인
- **port 번호**: `3000`이 port 번호. **같은 port에는 동시에 다른 서버를 열 수 없음**
- 종료: 해당 터미널 창을 닫아야 서버가 꺼짐 (안 끄면 다음 작업 시 같은 port 사용 불가)

> **Tip – localhost**: localhost는 "내 집 주소". 남이 함부로 접근 불가. **남에게 배포하려면 도메인을 구매해 배포**해야 함 (배포는 다음 주)

## 9. OpenAI API 연동 & `.env` 보안

### `.env`로 키 보호
- 해당 폴더 안에 `.env` 파일 생성 후 키 저장
  - 형식: `지정한_키이름=발급받은_KEY값`
  - 예: `OPENAI_API_KEY=sk-proj-...`
- 키 이름은 **공백 없이 언더바(`_`)** 사용
- `.env`에 두면 안전하며 **GitHub에 푸시되지 않음**

### Claude에게 키 위치 알려주기
```
OpenAI API 이용해서 실제 AI 응답을 담아줘
API Key : 폴더루트/.env에 있어
@single-server-specialist
```

## 10. Express.js 도입 & 마무리

### `.env` 사용 시 서버 실행 명령어 변경
- 기존: `node 폴더명/server.js`
- `.env` 사용: `cd 폴더명 && node --env-file=.env server.js`
  - `--env-file=.env` 옵션이 추가된 형태

### package.json 활용
- 명령어를 일일이 기억하기 어려움 → 외부 package 사용·명령어 정리를 위해 `package.json` 사용
- **Express**: Node 웹서버를 더 빠르게 만들어주는 외부 package
- 예시:
  ```json
  "scripts": {
    "start": "node server.js",
    "dev": "node --watch server.js"
  },
  "dependencies": {
    "express": "^4.21.2"
  }
  ```
  - `dependencies`의 `express` 버전 표기 → 외부 패키지 사용 의미

### script로 서버 실행하는 3가지 방법
1. `scripts`에 서버 실행 명령어를 적어둠
2. 해당 script에 **우클릭 → run script** 클릭
3. **단축 명령어**: `package.json`이 있는 폴더로 이동(`cd 03`) 후
   - `npm run dev` 또는 `npm run start`
   - (`start`/`dev` 중 어떤 명령을 쓸지만 지정)
