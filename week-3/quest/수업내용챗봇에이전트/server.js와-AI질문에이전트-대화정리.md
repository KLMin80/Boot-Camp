# server.js와 "AI 질문 에이전트" 구조 — Claude와의 대화 정리

- **날짜:** 2026-06-14
- **참여자:** 나(수강생) ↔ Claude
- **한 줄 요약:** `server.js`가 웹앱에서 하는 역할을 이해하고, "내 튜터(lesson-qa-tutor)에게 질문하는 웹앱"을 만들려면 왜 `server.js`와 API 키가 필요한지(그리고 안 쓰는 방법까지)를 정리했다.

---

## 1. server.js는 무슨 역할을 하는 파일인가? (개념)

### 한 문장으로
> **server.js는 우리 프로젝트의 "접수 창구 + 백오피스 직원" 역할이다.** 브라우저의 모든 요청을 받아서, 직접 처리하거나 외부(OpenAI·날씨 API 등)에 대신 연락해 주고, 결과를 브라우저에 돌려준다.

### 카페에 빗대면

| 카페 비유 | server.js에서 |
|---|---|
| 손님(브라우저)이 카페에 들어옴 (`localhost:3000` 접속) | 요청(request)이 들어옴 |
| 직원이 메뉴판을 건네줌 | index.html을 내려줌 (정적 파일 서빙) |
| 손님이 "아메리카노" 주문 (`/api/greeting` 호출) | API 요청 |
| 직원이 음료를 만들어 가져다줌 | JSON 응답 반환 |
| 직원이 창고에서 원두를 꺼냄 (손님은 창고 못 들어감) | OpenAI·날씨 API를 대신 호출 (프록시) |

### 수업에서 server.js가 발전한 3단계

**1단계 — 파일 배달부** (`week-3\class\sever-test\01\server.js`)
```js
if (req.url === '/' || req.url === '/index.html') {
  fs.readFile(path.join(__dirname, 'index.html'), ...) // 파일을 읽어서
  res.end(data);                                        // 그대로 브라우저에 보냄
} else if (req.url === '/api/hello') {
  res.end(JSON.stringify({ message: '안녕하세요! 서버에서 보낸 응답입니다.' }));
}
```
브라우저는 파일을 직접 못 여니까 **server.js가 대신 배달**해 준다.

**2단계 — 로직을 서버에서 처리** (`week-3\class\sever-test\02\server.js`)
```js
function getGreeting(hour) {
  if (hour >= 5 && hour < 12) return "좋은 아침이에요";
  if (hour >= 12 && hour < 18) return "좋은 점심예요";
  ...
}
```
주석에 *"원래 index.html에 있던 로직을 그대로 서버로 옮겨왔습니다"* 라고 적혀 있다. 브라우저가 하던 계산을 **서버가 대신 하고 결과만 내려주는** 구조로 바꾼 단계.

**3단계 — 외부 API 프록시 (가장 중요)** (`my-mental-assistant`, `quest\AI별명생성기` 등)

### 왜 브라우저에서 직접 OpenAI를 안 부르고 server.js를 거치나?
```
나쁜 방식:   브라우저 → OpenAI            (API 키가 브라우저 코드에 그대로 노출! ❌)
올바른 방식:  브라우저 → server.js → OpenAI  (키는 server.js 안에서만, .env에 숨김 ✅)
```
브라우저 코드(index.html·client.js)는 개발자 도구로 누구나 볼 수 있어서, 키가 거기 있으면 도난당한다. 이 중간 역할을 **프록시(Proxy)** 라고 부른다.

### 정리 — server.js가 하는 일 4가지
1. **정적 파일 서빙** — index.html·client.js 같은 파일을 건네줌
2. **API 엔드포인트 운영** — `/api/...` 요청을 받아 JSON으로 응답
3. **외부 API 프록시** — OpenAI·날씨 API를 대신 호출, **키는 브라우저에 절대 안 내려보냄**
4. **포트에서 대기** — `server.listen(3000, ...)` 으로 켜진 채 요청을 기다림

## 2. "나만의 질문 에이전트"를 만들려면? — 왜 server.js가 또 필요한가

목표: **브라우저에서 내 튜터(`lesson-qa-tutor.md`)에게 질문하고 답을 받는 웹앱.**

핵심 사실: `lesson-qa-tutor`는 **Claude Code 안에서 도는 에이전트**라, 브라우저에서 열리는 `index.html`이 **직접 호출할 수 없다.** 그래서 중간에 **`server.js` 프록시**가 한 번 더 필요하다.

```
브라우저(index.html)  →  server.js  →  (튜터/LLM)  →  답  →  브라우저
```

## 3. 그런데 왜 API(키)가 필요하지? 내 컴퓨터가 답하는 거 아냐?

> **내 질문:** server.js가 내 컴퓨터(=실질적으로 lesson-qa-tutor)에게 전달해서 답을 받아오는 거라면, 왜 OpenAI/Claude API 키가 필요하지? 필요하다면 그건 외부가 아니라 **내 서버가 발급한 키**여야 하는 거 아냐?

### 핵심 오해: `lesson-qa-tutor.md`는 "두뇌"가 아니라 "지시서"다
`lesson-qa-tutor.md`는 스스로 문장을 만들어내는 프로그램이 **아니다.** *"이런 말투로, 이 수업파일을 근거로 답해라"* 라고 적힌 **지시서(설정 파일)** 일 뿐이다. 실제로 문장을 **생성하는 두뇌 = LLM(거대 언어모델)** 이고, 그 두뇌는 **내 컴퓨터가 아니라 Anthropic/OpenAI의 서버**에 있다.

### 실제 흐름 (키가 필요한 지점)
```
브라우저 ──질문──▶ server.js (내 컴퓨터)          ← 여기까지 내 통제, 키 필요 ❌
                       │
                       └──인터넷 건너──▶ LLM 두뇌 (Anthropic/OpenAI 서버)
                                              · 여기서 진짜 답을 생성
                                              · 남의 회사 유료 서비스 → 키로 인증·과금 🔑
                       ┌──생성된 답──────────────┘
브라우저 ◀──답변── server.js
```

### 두 질문에 대한 답
1. **왜 API가 필요해?** `browser → server.js → browser` 구간은 전부 내 컴퓨터 안이라 **어떤 키도 필요 없다.** 키가 필요한 건 server.js가 **바깥의 LLM(남의 서버)** 에 손을 뻗는 그 순간뿐.
2. **내 서버가 발급한 키여야 하지 않나?** 아니다. 키의 정체는 *"내 컴퓨터 접근권"* 이 아니라 **"남의 두뇌(LLM)를 빌려 쓰는 이용권"** 이다. 그러니 그 두뇌를 가진 회사(OpenAI/Anthropic)가 발급한다.
   - (참고) "내 서버가 발급하는 키"는 방향이 반대인 개념이다. 내 server.js가 *"아무나 쓰지 마"* 하고 막고 싶을 때 내가 토큰을 만들어 검사할 수 있다. 하지만 그건 **답 생성용이 아니라 내 서버 보호용**(선택)이라 목적이 다르다.

### 외부 키 없이 "진짜 내 컴퓨터가 답하게" 하는 길 — 두뇌를 내 컴퓨터에 두면 된다
1. **로컬 LLM (Ollama 등)** — 언어모델을 내 PC에 설치. **외부 API 키 0개.** `server.js → localhost:11434`. 내 직관에 가장 가까움. 대신 PC 사양 필요, 답 품질은 GPT/Claude보다 낮음.
2. **진짜 `lesson-qa-tutor`를 `claude -p`로 호출** — Claude Code에 **이미 로그인된 인증을 그대로** 사용 → **새 API 키 발급 불필요.** 로컬 수업파일을 자동 근거로 삼아 가장 충실. 단, 답 자체는 외부(Anthropic의 Claude)가 만들고 응답이 좀 느림.
3. **OpenAI/Claude API** — 새 키 하나 발급해 `.env`에. 품질 최고·셋업 간단, 소액 과금.

## 4. 최종 도식 — `lesson-qa-tutor.md`의 역할까지 넣어서

### 비유 — "창구 + 매니저 + 알바 선생님"
- 🧑‍💼 **알바 선생님 = LLM 두뇌 (Claude/OpenAI)** — 머리는 좋은데 우리 수업은 모름. 외부 소속이라 쓰면 **사용료(=API 키)** 가 나감.
- 📋 **업무 대본 = `lesson-qa-tutor.md`** — 선생님에게 쥐여주는 종이. *"너는 친절한 수업 튜터야. 이 수업노트를 근거로 한국어로 답해."* **이 종이는 스스로 말을 못 함.**
- 📚 **교재 = 수업파일(수업내용·코드)** — 답할 때 펴 보는 근거 자료.
- 🧑‍🔧 **매니저 = `server.js`** — 질문을 받아 **대본 + 교재 + 질문을 한 봉투에** 챙겨 선생님께 전달하고, 답을 손님께 가져다줌.
- 🪟 **창구 = 브라우저(index.html)** — 손님이 질문을 적고 답을 보는 곳.

### 그림
```
①  [브라우저 / index.html]
        │   질문을 입력해서 보냄
        ▼
②  [server.js — 내 컴퓨터의 "매니저"]      ← 여기까지 전부 내 컴퓨터, API 키 필요 없음 ❌
        │
        │   질문 하나에 아래 셋을 "한 봉투"로 묶음:
        │     • lesson-qa-tutor.md    = 📋 지시서(대본): "이 말투로, 이 자료를 근거로 답해라"
        │     • 수업파일(노트·코드)    = 📚 근거 자료
        │     • 사용자 질문            = 실제 물어본 내용
        ▼
③  [LLM 두뇌 / Claude·OpenAI]            ← 바깥(남의 서버), 🔑 API 키로 인증·과금 ✅
        │   봉투를 읽고 '지시서대로' 진짜 답을 생성
        ▼
④  [server.js]  생성된 답을 받아서
        ▼
⑤  [브라우저]  화면에 답을 표시
```

### 역할 한눈에 — "키가 필요한가?"

| 구성요소 | 비유 | 하는 일 | 🔑 키? |
|---|---|---|:---:|
| 브라우저(index.html) | 창구 | 질문 입력·답 표시 | ❌ |
| **server.js** | 매니저 | 대본+교재+질문을 묶어 두뇌에 전달, 답 회수 | ❌ (내 컴퓨터) |
| **lesson-qa-tutor.md** | 업무 대본 | "이 말투·이 근거로 답해라" 규칙. **스스로 답 못 함** | ❌ (그냥 텍스트) |
| 수업파일(노트·코드) | 교재 | 답의 근거 자료 | ❌ |
| **LLM(Claude/OpenAI)** | 똑똑한 선생님 | **진짜 답을 생성** | ✅ (바깥 유료) |

### 핵심 한 줄
> `lesson-qa-tutor.md`는 **두뇌가 아니라 "두뇌에게 건네는 지시서"** 다. 그래서 키와 무관한 종이일 뿐이고, **🔑 키는 오직 ③ '바깥 두뇌(LLM)'를 쓸 때만** 필요하다.
