<a id="0-안전경고"></a>
## 1. ⚠️ 먼저 읽는 안전·법적 경고 — **건너뛰지 마세요**

자동매매는 **실제 내 돈이 자동으로 사고 팔리는** 프로그램입니다. 버그 하나가 곧 손실입니다. 아래는 타협 대상이 아닙니다.

1. **무조건 "모의투자"부터.** 키움/한투 REST API 모두 **모의투자(모의도메인)** 를 제공합니다. 실전 도메인은 전 단계 전략이 몇 주간 모의로 검증된 뒤에만 켭니다. 이 문서의 모든 프롬프트는 **모의투자 기본값**으로 만들라고 지시합니다.
2. **API 키/시크릿은 절대 코드·깃허브에 넣지 않습니다.** `.env` 파일에만 두고 `.gitignore`에 `.env`를 등록합니다. 커밋 전 키가 들어갔는지 스캔합니다. (키 유출 = 계좌 탈취)
3. **안전장치를 코드보다 먼저.** 일일 최대 주문 횟수·금액 한도, 종목당 최대 비중, "실전모드일 때 주문 전 텔레그램 승인" 옵션을 처음부터 넣습니다.
4. **소액·검증.** 실전 전환 후에도 최소 금액으로 시작하고, 로그로 "내 전략이 의도대로 도는지"를 며칠 지켜본 뒤 늘립니다.
5. **책임은 나에게.** 이 문서·강의·AI 누구도 수익을 보장하지 않습니다. 투자 손실은 전적으로 본인 책임입니다. 자동매매 관련 증권사 약관·세금·과당매매 규정도 스스로 확인하세요.
6. **"AI가 짜준 코드"를 그대로 믿지 않습니다.** 특히 **주문 수량·가격·매수/매도 방향**은 사람이 눈으로 두 번 확인합니다. AI는 매도(01)와 매수(02) 코드를 바꿔 쓰는 실수도 합니다.

> 💡 이 6개를 지키면 최악의 사고(잘못된 대량 주문, 키 유출)는 거의 다 막힙니다.

<a id="아키텍처"></a>
## 2. 이 앱이 결국 뭘 하는 물건인가 (아키텍처)

강의가 만드는 것은 **"내 PC에서 24시간 돌아가는 개인 트레이딩 비서"** 입니다. 폰으로 텔레그램 톡을 보내 명령하고, 봇이 증권사 API로 주문·조회하고, 전략 엔진이 알아서 감시·매매합니다.

```
   [ 내 폰/PC 텔레그램 ]  ←── 톡(명령·알림·차트) ──→  [ 텔레그램 봇 ]
                                                         │
                                                         ▼
        ┌──────────────  내 PC에서 돌아가는 매매 프로그램  ──────────────┐
        │  ① 명령 파서   (원격 명령 체계 / 자연어·음성 → 구조화 명령)      │
        │  ② 전략 엔진   (스탑로스·골든크로스·트레일링·돌파·분할·그리드)   │
        │  ③ 주문 실행기 (안전장치·쿨다운·종목제한 통과 후 주문)          │
        │  ④ 로거/리포터 (일일 로그 → 엑셀 → AI 분석 → 톡 전송)          │
        └──────────────┬───────────────────────────┬─────────────────┘
                       │ REST API                  │ WebSocket(실시간)
                       ▼                           ▼
             현재가·잔고·주문·순위        체결통보·실시간시세·조건검색식·VI발동
                       └───────────────┬───────────┘
                                       ▼
                        [ 증권사 서버 — 모의투자 / (검증 후)실전 ]
```

**핵심 데이터 흐름 3가지**
- **REST**: 요청→응답 방식. 현재가 조회, 잔고 조회, 주문 넣기 등 "내가 물어보는" 것.
- **WebSocket**: 서버가 실시간으로 밀어주는 것. 체결통보, 실시간 시세, **영웅문 조건검색식에 종목이 포착됨**, VI 발동.
- **Telegram**: 사람 ↔ 프로그램. 명령 넣고 알림 받는 창구.

<a id="준비물"></a>
## 3. 시작 전 준비물 체크리스트

프롬프트를 던지기 전에 **사람이 직접** 준비해야 하는 것들입니다. (AI가 대신 못 해줌)

- [ ] **증권사 계좌 + REST API 신청** — 키움증권 기준: 키움 REST API(오픈API) 사용 신청 → **앱키(App Key) / 시크릿(App Secret)** 발급. **모의투자 계좌**도 함께 개설/신청. (한국투자증권 KIS Developers도 동일 구조라 대체 가능)
- [ ] **접속 IP 확인** — 일부 증권사 API는 등록된 IP에서만 호출됩니다. 집 인터넷은 보통 유동IP → *Phase 9*에서 해결.
- [ ] **텔레그램** — 앱 설치 → `@BotFather`로 새 봇 생성 → **봇 토큰** 확보. 내 **chat_id**도 확인(봇에게 아무 말이나 보낸 뒤 getUpdates로 확인, 이건 AI가 코드로 도와줌).
- [ ] **개발 도구** — 코드 에디터(Cursor 또는 VS Code + Claude). 이 문서의 프롬프트를 붙여넣을 곳.
- [ ] **런타임 설치** — Python 3.11+ (권장) 또는 Node.js 18+. (아래 스택 참고)
- [ ] **(선택) LLM API 키** — 자연어 명령·일일 AI 분석에 쓸 Gemini 또는 Claude API 키. *Phase 7·8*에서 사용.
- [ ] **영웅문(HTS) 또는 조건검색식** — 조건검색식 연동(*Phase 5*)을 쓰려면, HTS에서 내 조건검색식을 미리 하나 만들어 저장.

<a id="스택"></a>
## 4. 기술 스택 결정 (이 문서의 기본 선택)

이 도메인(국내 주식 자동매매 + 웹소켓 + 텔레그램)은 **Python** 생태계가 가장 두껍습니다. 그래서 **기본값은 Python**으로 잡았습니다. 단, 프롬프트는 언어에 크게 의존하지 않게 써두었으니 **Node.js가 더 편하면 스택만 바꿔서** 그대로 쓰면 됩니다.

| 역할 | 기본 선택(권장) | 대안 |
|---|---|---|
| 언어/런타임 | **Python 3.11+** | Node.js 18+ |
| 증권사 API | **키움 REST API + WebSocket** (모의투자) | 한국투자증권 KIS Developers |
| 텔레그램 | `python-telegram-bot` | `node-telegram-bot-api` |
| HTTP/WS | `requests` / `websockets` | `axios` / `ws` |
| 스케줄/루프 | `asyncio` | `node-cron` |
| 설정/비밀 | `.env` (`python-dotenv`) | `.env` (`dotenv`) |
| 로그/리포트 | `pandas` → 엑셀(`openpyxl`) | `exceljs` |
| 자연어/AI분석 | Gemini 또는 Claude API | — |

> ⚠️ **API 엔드포인트·필드명은 이 문서에 하드코딩하지 않았습니다.** 증권사 API는 도메인 주소·TR코드·파라미터명이 수시로 바뀌고, 문서에 박아두면 낡습니다. 그래서 모든 프롬프트는 **"AI가 키움 REST API 공식 문서를 근거로 최신 스펙을 확인해서" 구현**하도록 지시합니다. AI에게 던질 때 **공식 API 문서 링크나 PDF를 함께 첨부**하면 정확도가 확 올라갑니다.

## 5. Phase 0 — 마스터 컨텍스트 프롬프트 (딱 한 번)

가장 먼저, **딱 한 번** 던지는 "이 프로젝트가 뭔지 + 규칙"을 세우는 프롬프트입니다. Cursor면 `.cursorrules`(또는 Project Rules), Claude Code면 `CLAUDE.md`에 저장해두면 이후 모든 대화에서 AI가 이 규칙을 기억합니다.

```text
너는 국내 주식 "개인용 자동매매 프로그램"을 나와 함께 만드는 시니어 개발 파트너다.
나는 코딩 초보다. 전문용어는 풀어서 설명하고, 한 번에 한 단계씩만 진행해라.

[프로젝트 목표]
- 내 PC에서 24시간 돌아가는 개인 트레이딩 비서.
- 텔레그램으로 명령/알림을 주고받고, 증권사 REST+WebSocket API로 조회·주문·실시간감시.
- 스탑로스·골든크로스·트레일링스탑·돌파매수·분할/그리드 등 매매 전략을 내가 켜고 끌 수 있다.

[기술 스택]
- 언어: Python 3.11+ (내가 Node를 원하면 그때 바꾼다)
- 증권사: 키움증권 REST API + WebSocket, "모의투자" 도메인을 기본값으로 한다.
- 텔레그램: python-telegram-bot
- 비밀값: 전부 .env 로 관리하고 .gitignore에 .env를 넣는다. 코드에 키를 절대 하드코딩하지 마라.

[반드시 지킬 안전 규칙 — 최우선]
1. 기본은 모의투자다. 실전(real) 도메인은 내가 명시적으로 "실전으로 바꿔줘"라고 할 때만, 그리고 환경변수 IS_REAL=true 일 때만 활성화한다.
2. 주문 함수에는 항상 안전장치를 건다: 일일 최대 주문 횟수/금액 한도, 종목당 최대 매수금액, 그리고 IS_REAL=true일 때는 "주문 직전 텔레그램으로 승인 요청" 옵션.
3. 매수/매도 방향, 수량, 가격은 절대 헷갈리지 마라. 주문 코드에는 사람이 읽는 한글 주석으로 "이건 매수/매도"를 반드시 달아라.
4. 증권사 API의 엔드포인트·TR코드·파라미터명은 네 기억에 의존하지 말고, 내가 준 공식 문서/링크를 근거로 확인해라. 불확실하면 추측하지 말고 나에게 "공식 문서의 이 부분을 확인해달라"고 물어라.
5. 모든 외부 호출(API·웹소켓·텔레그램)은 실패·재시도·타임아웃을 처리해라. 특히 토큰 만료와 웹소켓 끊김 재접속.

[일하는 방식]
- 큰 기능은 작은 단계로 쪼개서, 매 단계 "무엇을/왜" 한 줄 설명 → 코드 → "이렇게 테스트해봐" 순으로 준다.
- 폴더 구조와 파일 역할을 먼저 제안하고 내 확인을 받은 뒤 코드를 쓴다.
- 내가 "OK"라고 하기 전엔 다음 단계로 넘어가지 마라.

이 규칙을 이해했으면, 먼저 이 프로젝트의 폴더/파일 구조와 .env 에 들어갈 항목 목록만 제안해줘. 코드는 아직 쓰지 마.
```

> 이 프롬프트를 던지면 AI가 폴더 구조와 `.env` 항목을 제안합니다. 확인하고 "OK, 이대로 가자"라고 답한 뒤 Phase 1로 갑니다.

<a id="phase1"></a>
## 6. Phase 1 — 프로젝트 뼈대 + 증권사 인증(토큰)

> 강의 매핑: 6~9강(개발환경·앱키·부품 설치). **가장 중요한 첫 관문.** 여기만 넘으면 뒤는 쉽습니다.

```text
[Phase 1: 프로젝트 뼈대와 증권사 인증]

1) 아까 합의한 폴더 구조로 빈 프로젝트를 만들어줘. 최소한 이렇게:
   - config: .env 로더 (앱키/시크릿, 모의/실전 토글 IS_REAL, 텔레그램 토큰/챗ID, 각종 한도값)
   - broker: 증권사 API 클라이언트 (인증/조회/주문을 여기 모은다)
   - main: 프로그램 진입점
   - .env.example (실제 값 없이 키 이름만), .gitignore(.env 포함), requirements.txt

2) 증권사 "접근토큰(access token)" 발급 기능부터 만들어줘.
   - 키움 REST API 기준: 앱키/시크릿으로 OAuth 토큰을 발급받고, 만료 시간을 저장해뒀다가 만료 전에 자동 재발급하는 구조로.
   - 모의투자 도메인을 기본으로, IS_REAL 값에 따라 실전 도메인으로 스위치되게.
   - 토큰 발급 성공/실패를 콘솔에 명확히 찍어줘.
   ※ 엔드포인트와 요청 형식은 내가 붙여줄 키움 REST API 공식 문서를 근거로 해줘. (문서 없이 추측 금지)

3) 잘 됐는지 확인용으로, 실행하면 "토큰 발급 성공 + 남은 유효시간"만 출력하는 최소 테스트 스크립트를 줘.

한 번에 다 하지 말고, 먼저 1)폴더구조와 .env.example 부터 보여줘. 내가 OK 하면 2)로 가자.
```

**여기서 자주 막히는 것 (미리 알려주기용)**
- 앱키/시크릿을 **모의투자용/실전용 따로** 발급해야 하는 경우가 있습니다. 모의용 키인지 확인.
- 토큰 발급이 401/403이면 대개 (a)키 오타 (b)모의/실전 도메인 불일치 (c)IP 미등록. 이 3개를 AI에게 체크리스트로 시키세요.

## 7. Phase 2 — 조회 기능 (현재가·잔고·순위)

> 강의 매핑: 10강(간단한 조회), 23강(순위조회). 주문 전에 **읽기(조회)부터** 확실히. 조회는 사고가 안 나니까 여기서 API 감각을 익힙니다.

```text
[Phase 2: 조회 기능]
broker 클라이언트에 아래 조회 함수들을 하나씩 추가해줘. 각 함수는 결과를 깔끔한 파이썬 dict/리스트로 정리해서 반환하고, 실패 시 예외를 로깅하도록.

1) 현재가 조회: 종목코드(예: 삼성전자 005930)를 넣으면 현재가/등락률/거래량 반환.
2) 계좌 잔고 조회: 보유 종목 리스트(종목명·수량·평균단가·현재가·평가손익·손익률)와 예수금/주문가능금액 반환.
3) 순위 조회: 거래량 상위 / 등락률 상위 같은 순위 데이터를 N개 반환.

각 함수마다 "이렇게 한 줄 호출해서 결과를 print 해봐" 하는 사용 예시도 같이 줘.
엔드포인트/TR은 내가 준 공식 문서 기준으로. 필드명이 불확실하면 나에게 문서 위치를 물어봐.

먼저 1)현재가 조회 하나만 만들어서 내가 삼성전자로 테스트해볼게. 되면 2),3) 가자.
```

## 8. Phase 3 — 텔레그램 원격 명령 체계

> 강의 매핑: 11강(텔레그램 연동), 15강(원격 명령 체계), 20강(설정·예약·도움말). 이제 **폰에서 조종**합니다.

```text
[Phase 3: 텔레그램 원격 명령 체계]
python-telegram-bot 으로 내 봇을 붙이고, 톡으로 명령을 내리는 체계를 만들어줘.

1) 봇 기본 연결: .env의 봇 토큰/내 chat_id 사용. 내 chat_id에서 온 메시지만 처리(다른 사람 차단).
2) 명령 라우터: "/도움말", "/잔고", "/현재가 005930", "/시세순위" 같은 슬래시 명령을 Phase 2 조회 함수에 연결해서 결과를 톡으로 예쁘게 보내줘.
3) /도움말: 지금 쓸 수 있는 모든 명령 목록을 자동으로 보여주게.
4) 프로그램이 켜지면 "비서 가동 시작" 알림을, 에러가 나면 에러 요약을 나에게 톡으로 보내는 알림 유틸도.

보안: 등록되지 않은 chat_id는 무시하고 로그만 남겨. 명령 실행 결과는 항상 톡으로 피드백.
먼저 1),2)의 /잔고, /현재가 까지만 되게 만들어줘.
```

> 강의의 "예약" 기능(특정 시각에 명령 실행)이 필요하면 이 단계 끝에 "특정 시각/장 시작·마감에 자동 실행되는 예약 명령도 추가해줘"를 덧붙이세요.

## 9. Phase 4 — 매수·매도 주문 실행 ⚠️

> 강의 매핑: 19강(종목 사고팔기). **여기서부터 실제 주문**입니다. 안전장치를 코드보다 먼저.

```text
[Phase 4: 매수/매도 주문 — 안전장치 우선]
실제 주문 기능을 만들되, "안전장치를 먼저" 만들고 그 다음에 주문을 붙여줘.

1) 안전장치 모듈 먼저:
   - 일일 주문 횟수 한도 / 일일 총매수금액 한도 / 종목당 최대 매수금액 (.env로 설정)
   - 종목 화이트리스트·블랙리스트(살 수 있는/못 사는 종목 제한)
   - 쿨다운: 같은 종목을 방금 매매했으면 N초/분간 재주문 금지(중복·과당매매 방지)
   - IS_REAL=true면 주문 직전 텔레그램으로 "○○ 10주 시장가 매수할까요? [예/아니오]" 승인 요청
   위 검사를 모두 통과해야만 실제 주문 함수가 호출되게.

2) 주문 함수:
   - 매수/매도, 시장가/지정가를 지원. 함수 시그니처와 파라미터에 한글 주석으로 매수/매도를 명확히.
   - 모의투자 도메인 기본. 주문 성공 시 주문번호를, 실패 시 사유를 반환하고 텔레그램으로 결과 통보.

3) 텔레그램 명령 연결: "/매수 005930 10 시장가", "/매도 005930 전량" 같은 명령. 단, 위 안전장치를 반드시 거치게.

⚠️ 매수(주문구분 코드)와 매도 코드를 절대 바꾸지 마. 공식 문서에서 주문 파라미터를 재확인하고, 헷갈릴 여지가 있으면 나에게 먼저 물어봐.
먼저 1)안전장치 모듈부터 만들고 테스트한 뒤, 내 OK를 받고 2)주문으로 가자.
```

> 🧪 **모의투자로 첫 주문을 넣고, 증권사 앱/HTS에서 그 주문이 실제로 들어갔는지 눈으로 대조**하세요. 이 대조를 통과하기 전엔 전략(Phase 6)으로 넘어가지 마세요.

## 10. Phase 5 — 실시간 웹소켓 (체결통보·조건검색식·VI)

> 강의 매핑: 26~27강(조건검색식 연동 — "여기가 고비!"), 28강(트레일링/웹소켓), 39강(VI 감지). 실시간의 심장.

```text
[Phase 5: 실시간 WebSocket]
증권사 WebSocket에 붙어서 서버가 실시간으로 밀어주는 데이터를 받는 구조를 만들어줘.

1) 웹소켓 연결 관리자: 접속·로그인(토큰)·자동 재접속(끊기면 지수백오프로 재연결)·하트비트. 끊김/재접속을 텔레그램으로 알림.
2) 실시간 체결통보 구독: 내 주문이 체결되면 즉시 받아서 "○○ 10주 체결 @ 가격" 을 톡으로.
3) 실시간 시세 구독: 관심종목의 실시간 현재가를 받아 전략 엔진이 쓸 수 있게 콜백/큐로 전달.
4) 조건검색식 연동: 영웅문(HTS)에서 내가 만들어둔 조건검색식 목록을 불러오고, 특정 조건식에 "새 종목이 포착"되면 실시간으로 받아서 → 내가 지정한 자동매매 동작(예: 자동 매수 후보로)에 연결.
5) VI(변동성완화장치) 발동 감지: 관심종목에 VI가 걸리면 감지해서 톡 알림 + 전략에서 활용 가능하게.

각 구독은 켜고 끌 수 있게. 엔드포인트/전문 포맷은 내가 준 공식 웹소켓 문서를 근거로.
가장 먼저 1)연결관리자 + 2)체결통보 까지만. 모의로 주문 넣어서 체결 톡이 오는지 확인하고 다음.
```

**"여기가 고비" 대응(26·27강):** 조건검색식이 계속 조회 안 되면 AI에게 이렇게 시키세요 →
`"조건검색식이 조회되지 않아. 웹소켓 로그인 전문, 조건식 목록 요청 전문, 실시간 등록 전문을 순서대로 콘솔에 그대로 찍어서 어디서 실패하는지 같이 진단하자."`

## 11. Phase 6 — 전략 엔진 (자동매매의 두뇌)

> 강의 매핑: 21강(스탑로스), 24강(골든/데드크로스), 28강(트레일링 스탑), 40강(돌파매수), 41강(분할), 42강(그리드).

전략은 **하나씩** 붙입니다. 각 전략은 **① 감시 → ② 조건 충족 → ③ 안전장치 통과 → ④ 주문(Phase 4)** 흐름을 공유합니다. 먼저 공통 뼈대를 만들고, 그 위에 전략을 얹으세요.

### 6-0. 전략 엔진 공통 뼈대 (먼저)

```text
[Phase 6-0: 전략 엔진 공통 뼈대]
개별 전략을 얹기 전에 공통 뼈대를 만들어줘.
- Strategy 라는 공통 인터페이스: on_price(실시간 시세) / on_fill(체결) 콜백을 받고, 조건 충족 시 Phase 4의 "안전장치→주문" 경로로만 주문.
- 전략마다 대상 종목·파라미터를 .env나 설정파일에서 읽고, 텔레그램 "/전략목록", "/전략켜기 스탑로스", "/전략끄기 스탑로스"로 on/off.
- 모든 전략 동작(감시 시작/조건 충족/주문/스킵 사유)을 로그와 텔레그램에 남겨서 "왜 샀는지/안 샀는지"를 항상 알 수 있게.
이 뼈대만 먼저 만들고, 전략은 내가 하나씩 요청할게.
```

### 6-1. 스탑로스 (손절)

```text
[전략: 스탑로스]
보유 종목이 "매입가 대비 -X%"(설정값, 예: -3%) 이하로 떨어지면 자동으로 시장가 매도해서 손실을 끊는 전략을 만들어줘.
- 실시간 시세 또는 주기적 조회로 보유종목 손익률 감시.
- 트리거 시: 안전장치 통과 → 전량(또는 설정 비율) 매도 → 텔레그램 통보("스탑로스 발동: ○○ -3.2%에 매도").
- 종목별로 다른 손절선을 줄 수 있게.
```

### 6-2. 골든크로스 / 데드크로스 (이동평균 교차)

```text
[전략: 골든크로스·데드크로스]
단기 이동평균선이 장기 이동평균선을 "위로 뚫으면(골든크로스) 매수", "아래로 뚫으면(데드크로스) 매도"하는 전략.
- 이동평균 기간은 설정(예: 단기 5, 장기 20). 분/일봉 중 무엇을 쓸지 파라미터로.
- 과거 시세(캔들)를 조회해 이평선을 계산하고, 새 봉마다 교차 여부 판정.
- 골든크로스 → 설정 금액만큼 매수, 데드크로스 → 보유분 매도. 각각 안전장치 통과 후 실행하고 텔레그램 통보.
- "직전 봉에서 이미 교차 처리했는데 또 사는" 중복을 막아줘(상태 저장).
```

### 6-3. 트레일링 스탑 (고점 추적 손절)

```text
[전략: 트레일링 스탑]
매수 후 "최고가를 계속 추적"하다가, 그 고점 대비 -Y%(예: -2%) 떨어지면 매도해서 이익을 지키는 전략.
- 종목별로 매수 후 갱신되는 최고가를 저장하고, 실시간 시세로 (현재가 / 보유중 최고가) 비교.
- 고점 대비 하락률이 설정값 도달 시 매도 → 텔레그램 통보("트레일링스탑: 고점 대비 -2%, ○○ 매도").
- 실시간성이 중요하니 가능하면 웹소켓 시세 사용. HTTP 폴링 대비 장단점을 주석으로 설명해줘.
```

### 6-4. 돌파매수 (Breakout)

```text
[전략: 돌파매수]
특정 기준가(예: 당일 고가, 전일 고가, 지정 가격, 또는 조건검색식 포착가)를 "위로 돌파"하면 매수하는 전략.
- 기준가 계산 방식을 파라미터로(전고점/박스권 상단/사용자 지정).
- 돌파 확인 시 매수(안전장치 통과) + 통보. 돌파 직후 되돌림 대비 스탑로스/트레일링과 함께 쓰도록 연계 옵션.
- (선택) Phase 5 조건검색식에 포착된 종목을 돌파매수 후보로 자동 편입.
```

### 6-5. 분할 트레이딩 (분할 매수/매도)

```text
[전략: 분할 트레이딩]
한 번에 다 사지 않고 "여러 번 나눠서" 매수/매도하는 전략.
- 총 목표금액을 N회로 분할, 가격이 일정 % 내려갈 때마다 1회분씩 추가 매수(물타기형) 또는 올라갈 때마다 분할 매도(익절형) 중 선택.
- 회차·간격·1회분 금액을 설정. 몇 회차까지 진행됐는지 상태 저장하고 텔레그램으로 진행상황 보고.
- 최대 회차/최대 투입금액 한도를 안전장치와 연동.
```

### 6-6. 그리드 트레이딩 (격자 매매)

```text
[전략: 그리드 트레이딩]
가격 구간을 "격자(그리드)"로 나눠, 한 칸 내려가면 매수·한 칸 올라가면 매도를 반복해 횡보장에서 수익을 내는 전략.
- 상단가/하단가/격자 개수(또는 격자 간격)와 칸당 주문수량을 설정.
- 각 격자선 도달 시 매수/매도, 체결되면 반대편 격자에 대기 주문을 다시 거는 식으로 순환.
- 현재 격자 상태(어느 칸에 얼마 걸려있는지)를 저장하고 /그리드상태 로 조회. 상단/하단 이탈 시 처리 규칙도 정해줘.
```

> 🧭 전략마다 **모의투자에서 며칠 관찰 → 로그로 "의도대로 도는지" 확인 → 파라미터 조정**을 반복하세요. 여러 전략을 동시에 켤 땐 같은 종목에 충돌(한쪽은 사고 한쪽은 파는)나지 않게 AI에게 "전략 간 우선순위·충돌 방지 규칙"을 명시적으로 요청하세요.

## 12. Phase 7 — 자연어·음성 명령 레이어

> 강의 매핑: 31강(자연어 명령), 32강(커맨드 세이프 옵션), 33강(음성 명령). 강의의 "어시스턴트 젬(Gemini Gem)"에 해당하는, **가장 '와우'한** 부분.

```text
[Phase 7: 자연어 명령]
슬래시 명령 말고, "사람 말"로 지시하면 알아듣는 레이어를 얹어줘.
- 텔레그램에 "삼성전자 100만원어치 사줘", "지금 잔고 얼마야?", "카카오 손절선 -5%로 걸어줘" 같은 자연어를 보내면,
  LLM(Gemini 또는 Claude API)이 이를 "구조화된 명령(JSON: action, 종목, 수량/금액, 옵션)"으로 변환하고,
  그 명령을 기존 안전장치→실행 경로로 태워줘. LLM이 직접 주문 API를 부르는 게 아니라, "명령 해석"만 하고 실행은 우리 검증된 코드가 하도록.
- 애매하면(종목 불명확·수량 없음 등) 실행하지 말고 되물어봐: "카카오(035720) 맞나요? 몇 주 살까요?"
- [커맨드 세이프 옵션] 설정으로: safe 모드면 해석된 명령을 실행 전 "이렇게 실행할게요 [확인]"으로 사람 승인, off면 바로 실행.
- 종목명→종목코드 매핑 테이블(자연어의 흔한 이슈)도 만들어서 "삼성전자"→005930 를 안정적으로 변환.

먼저 조회성 자연어("잔고 얼마야")부터 붙이고, 검증되면 주문성 자연어로 확장하자.
```

```text
[Phase 7+: 음성 명령 (선택)]
텔레그램 음성 메시지를 받으면 STT로 텍스트 변환 후 위 자연어 파이프라인에 태워줘.
음성은 오인식이 잦으니, 주문성 명령은 무조건 safe 모드(사람 승인)로 강제해줘.
```

## 13. Phase 8 — 일일 로그 + 엑셀 + AI 분석 + 차트 이미지

> 강의 매핑: 35강(일일 로그·AI 분석), 36강(차트 이미지 톡). 하루를 **복기**하는 기능.

```text
[Phase 8: 일일 로그 & AI 분석 & 차트]
1) 매매기록 로깅: 모든 주문/체결을 (시각·종목·매수매도·수량·가격·전략·손익)으로 남기고, 하루 단위로 엑셀(.xlsx) 파일로 저장. 날짜별 시트 또는 파일.

먼저 1)로깅+엑셀 저장부터. 며칠 데이터 쌓이면 2),3) 붙이자.
```

## 14. Phase 9 — 운영·배포 (유동IP·24시간·Git)

> 강의 매핑: 45강(유동IP), 46강(다른 컴퓨터 실행), 49~52강(API/REST/웹소켓/Git 개념).

```text
[Phase 9: 운영·배포]
1) 유동IP 대응: 증권사 API가 등록 IP만 허용한다면, 현재 공인IP를 감지해 (a)바뀌면 알림 (b)가능하면 API로 등록 IP 자동 갱신, 또는 고정IP/특정 실행환경 대안을 정리해줘.
```

<a id="팁"></a>
## 15. AI에게 말 거는 법 (프롬프트 팁)

수강평에 나온 "AI 멱살 잡고 갈궈가며 만든다"의 실전 요령입니다.

1. **한 번에 하나.** "전부 만들어줘" 대신 "이 함수 하나만". 작게 줄수록 정확합니다.
2. **에러는 통째로 붙여넣기.** 요약하지 말고 콘솔 에러 전문을 그대로 주고 "이 에러 원인과 고칠 코드"라고 하세요.
3. **"왜 이렇게 했는지" 물어보기.** 코드를 받으면 "이 부분 왜 이렇게 했어? 한 줄씩 설명해줘"로 이해하고 넘어가야 나중에 직접 고칠 수 있습니다.
4. **공식 문서를 근거로.** 증권사 API는 "네 기억 말고 이 문서 기준으로"라고 못박고, 가능하면 문서 링크/PDF를 첨부하세요.
5. **되돌아가기 쉽게.** 잘 되던 상태는 그때그때 Git 커밋(또는 프로젝트 복제). 강의도 "고비 실습 전 프로젝트 복제 필수"라고 강조합니다.
6. **안전 재확인 습관.** 주문 관련 코드를 받으면 항상 "이거 매수/매도, 수량, 가격 맞는지 다시 확인해줘"를 한 번 더.
7. **모의부터 끝까지.** "실전으로 바꿔줘"는 전략이 모의에서 충분히 검증된 다음에만.
