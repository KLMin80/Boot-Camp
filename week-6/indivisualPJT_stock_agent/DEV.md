# DEV.md - 개발 가이드

> **톡트레이더(TalkTrader)** — 내 PC에서 24시간 돌며, 내가 정한 규칙과 안전장치대로 국내 주식을 감시·매매하고 텔레그램으로 명령·알림을 주고받는 **개인용 규칙기반 자동매매 비서.**
> **Architecture: 기능별 모듈 분리 구조 (Python) — Option B** — `config` / `broker` / `telegram_bot` / `safety` / `strategy` / `main`. 실제 돈이 오가는 프로그램이라 **"안전장치를 주문 코드에서 물리적으로 분리"** 하는 것을 최우선으로 한 구조.
> 최종 확정일: 2026-07-11 · 근거: `MISSION.md`, `주식_자동매매_빌드_프롬프트.md`, `research.md`, `AUDIENCES.md`

## Requirements

MISSION.md의 v1(MVP) 핵심 기능. v1 = **KIS 모의투자로 "인증 → 조회 → 텔레그램 → 모의주문(+안전장치) → 스탑로스"까지 한 바퀴 무사고 완주.** 아래가 전부 동작하고 검증되면 v1 성공(=실전 전환 게이트 1단계 통과).

- [ ] **증권사 인증(토큰) — 모의 기본** *(Phase 1)* — 앱키/시크릿으로 접근토큰 발급, 만료 전 자동 재발급. `IS_REAL` 토글 구조만 갖추되 기본은 모의.
- [ ] **조회 기능** *(Phase 2)* — 현재가, 계좌 잔고(보유종목·평가손익·주문가능금액). 주문 전에 "읽기"부터 확실히.
- [ ] **텔레그램 원격 명령** *(Phase 3)* — 내 chat_id만 허용. `/잔고`, `/현재가 005930`, `/도움말` + 가동/에러 알림.
- [ ] **안전장치 (주문 코드보다 먼저)** *(Phase 4a)* — 일일 주문 횟수·금액 한도, 종목당 한도, 쿨다운, 화이트리스트, (실전 시)텔레그램 승인. 이 검사를 모두 통과해야만 주문 함수가 불린다.
- [ ] **모의 주문** *(Phase 4b)* — 안전장치 통과 후에만 모의 매수/매도. 결과는 텔레그램 통보. **증권사 모의 화면과 방향·수량·가격 눈으로 대조.**
- [ ] **전략 ①: 스탑로스(손절)** *(Phase 6-1)* — 보유 종목이 매입가 대비 -X%(설정값) 이하로 떨어지면 안전장치를 거쳐 자동 매도 + 텔레그램 통보. 웹소켓 없이 **주기적 조회**로 동작.
- [ ] **비밀값 보안** *(전 단계 관통)* — 앱키/시크릿/봇토큰은 `.env`에만, `.gitignore`에 등록, 커밋 전 시크릿 스캔.

## Non-goals (v1에서 하지 않을 것)

- **실전(진짜 돈) 전환** — *최종 목표이지만* v1 밖. "실전 전환 게이트"를 통과한 뒤 별도 단계로 진행. v1의 모든 코드 기본값은 모의.
- **남에게 배포 · 멀티유저 · 상업화** — 순수 개인용 1인 사용.
- **실시간 웹소켓 고급 기능** *(Phase 5)* — 조건검색식 연동·VI 감지 등. 스탑로스는 웹소켓 없이 주기 조회로 충분.
- **나머지 전략** *(Phase 6-2~)* — 트레일링·분할·그리드·골든크로스는 v1 완주 후 하나씩.
- **자연어·음성 명령** *(Phase 7)*, **일일 로그·엑셀·AI 분석·차트** *(Phase 8)*, **24시간 무인 재시작·유동IP 자동갱신** *(Phase 9)* — 모두 이후 고도화.

## Style

- **인터페이스는 텔레그램 텍스트뿐. 화면(UI) 없음.** 조회·주문·전략 on/off를 전부 톡 한 줄로.
- **콘솔 로그가 곧 계기판.** 개발 중엔 모든 주요 동작(토큰 발급/조회 결과/안전장치 통과·스킵 사유/주문 요청·결과)을 콘솔에 명확히 찍어 "지금 뭐 하는지"를 항상 눈으로 본다.
- **초보 기준: 한 번에 한 단계.** 함수마다 "무엇을/왜" 한 줄 설명 + 한글 주석. 코드를 받으면 "이 부분 왜 이렇게 했어?"로 이해하고 넘어간다.
- **안전 문구가 코드보다 먼저.** 주문 파라미터엔 `# 이건 매수` / `# 이건 매도`를 한글 주석으로 반드시 명시(AI가 매수/매도 코드를 바꿔 쓰는 실수 방지).
- **모의가 기본값.** 실전 도메인은 게이트 통과 + `IS_REAL=true` + 사용자의 명시적 "실전으로 바꿔줘"가 3개 다 있을 때만.

## Key Concepts (핵심 용어 — 초보용 풀이)

- **모의투자(Paper trading):** 가짜 돈으로 **진짜 API를 쓰는 연습장.** 코드 로직은 실전과 100% 같고 돈만 가짜. 이 프로젝트의 기본값이자 안전벨트.
- **실전 전환 게이트:** 모의에서 성공지표(아래 '성공 지표')를 **전부** 충족해야만 실제 돈을 투입할 수 있다는 관문. 이걸 건너뛰면 안 됨.
- **broker 추상화:** 증권사마다 다른 API 호출 방식을 `broker` 폴더 한 곳에 가둬, 나중에 키움·토스로 갈아끼울 때 **다른 코드는 안 건드리게** 하는 설계.
- **안전장치(Guard):** 주문이 증권사로 나가기 전 반드시 통과해야 하는 **검문소.** 한도·쿨다운·화이트리스트·(실전)승인을 여기서 검사. 하나라도 걸리면 주문은 취소되고 "왜 막혔는지" 로그·톡으로 통보.
- **스탑로스(손절):** "-X%면 무조건 판다"는 규칙. 사람이 감정 때문에 가장 못 하는 손절을 기계가 대신 실행해 손실이 눈덩이처럼 커지는 걸 막는다. v1의 첫 전략.
- **접근토큰 자동 재발급:** KIS 접근토큰은 일정 시간 뒤 만료된다. 만료되면 장중에 봇이 멈추므로, 만료 전에 미리 새 토큰을 받아두는 구조가 필요.
- **주기적 조회 감시:** 웹소켓(실시간 푸시) 없이, 몇 초/분마다 보유종목 손익률을 REST로 물어봐서 손절선 도달 여부를 확인하는 방식. v1 스탑로스엔 이걸로 충분.

## Open Questions (아직 정해야 할 것 · 미검증 가정 = 리스크)

- **프로젝트 이름** — '톡트레이더'로 확정? (후보: 매매비서·지킴봇·오토폴리오·스탑가디언·톡비서)
- **첫 감시 종목 & 손절선** — 스탑로스를 모의로 검증하려면 "어떤 종목을 -몇 %에 손절"할 초기값이 필요.
- **감시 주기** — 주기적 조회를 몇 초/분 간격으로? (너무 잦으면 API 호출 한도 부담, 너무 뜸하면 손절 지연.)
- **안전장치 한도 초기값** — 일일 최대 주문 횟수/금액, 종목당 최대 매수금액을 얼마로 시작할지. (모의라도 습관을 실전 기준으로.)
- **모의 앱키/시크릿의 모의 전용 여부** — KIS는 모의용/실전용 키를 구분해 발급하는 경우가 있으니 발급 시 "모의용"인지 반드시 확인.

## 선택된 개발 구조 — 왜 "기능별 모듈 분리(Option B)"인가

MISSION.md에서 **스택은 이미 확정**(Python 3.11+ / 한국투자증권 KIS / python-telegram-bot / .env)됐다. 그래서 여기서 정할 것은 **"코드를 어떤 폴더·모듈로 나눌 것인가"** 다. 세 가지를 비교했다.

### Option A — 단일 파일 스크립트 (`main.py` 하나에 전부)
- **장점:** 가장 빠른 시작. 초보가 전체 흐름을 한 화면에서 봄. 파일 이동 없음.
- **단점:** 인증·조회·안전장치·주문·전략이 **한 파일에 뒤섞임.** 실제 돈이 오가는 코드에서 "매수/매도 방향 실수"·"안전장치 빼먹고 주문" 위험이 커진다. 파일이 수백 줄 넘으면 관리 불가. MISSION의 "안전장치를 주문과 분리" 원칙과 충돌.
- **판정:** ❌ v1이라도 부적합. **돈 관련 코드는 분리 자체가 안전장치**라서.

### Option B — 기능별 모듈 분리 ✅ 추천
- **구성:** `config`(설정·비밀) / `broker`(증권사 API) / `telegram_bot`(명령·알림) / `safety`(안전장치) / `strategy`(전략) / `main`(진입점).
- **장점:**
  - MISSION의 3대 설계 요구(**안전장치 독립 · broker 추상화 · 증권사 교체 용이**)와 정확히 정렬.
  - 각 파일이 **한 가지 책임**만 져서, 초보도 "어디를 고칠지" 바로 안다.
  - `safety`가 독립 모듈이라 **주문 함수는 안전장치를 거치지 않고는 절대 호출 불가**하게 구조로 강제할 수 있다(최고의 안전장치).
  - 나중에 키움/토스 추가 → `broker` 폴더만, 전략 추가 → `strategy` 폴더만 손대면 됨.
- **단점:** 파일이 여러 개라 처음엔 오갈 곳이 많다(그러나 초보가 감당 가능한 6~7개 수준).
- **판정:** ✅ **채택.** "한 바퀴 완주"라는 v1 목표와 "안전 최우선" 원칙에 가장 잘 맞는다.

### Option C — 계층형 패키지 (클린 아키텍처: 인터페이스/구현 분리, 의존성 역전)
- **장점:** 여러 증권사·여러 전략을 대규모로 동시에 굴릴 때 유리. 테스트 격리 최고.
- **단점:** 코딩 초보 + v1 완주가 목표인 지금은 **오버엔지니어링.** 추상 계층 설계에 발목 잡혀 정작 "완주"를 못 할 위험. Option B에서 필요할 때 자연스럽게 진화시킬 수 있다.
- **판정:** ❌ 지금은 과함. B로 시작해 필요해지면 그때 C로.

> **한 줄 결론:** 실제 돈이 자동으로 사고 팔리는 프로그램에서 가장 값싼 보험은 **"안전장치 코드를 주문 코드에서 떼어놓는 것"**. 그래서 Option B다.

## 폴더 / 파일 구조 (Option B 구현)

```
talktrader/
├─ .env                    # 🔒 비밀값: 앱키·시크릿·봇토큰·chat_id·한도값 (git 제외)
├─ .env.example            # 값 없이 "키 이름"만 적은 견본 (커밋용)
├─ .gitignore              # .env, logs/, __pycache__/ 제외
├─ requirements.txt        # 의존성 목록
├─ main.py                 # 진입점: 텔레그램 봇 시작 + 스탑로스 감시 루프 가동
├─ config/
│   └─ settings.py         # .env 로드, IS_REAL 토글, 도메인 선택, 한도값 상수 모음
├─ broker/
│   └─ kis_client.py       # KIS 인증(토큰 발급·자동 재발급)·조회·주문 — 증권사 의존 코드를 여기만 가둠
├─ telegram_bot/
│   └─ bot.py              # 명령 라우터(/잔고·/현재가·/매수…)·알림 유틸·chat_id 인증
├─ safety/
│   └─ guard.py            # 🚧 안전장치: 한도·쿨다운·화이트리스트·(실전)승인 검문소
├─ core/
│   └─ order.py            # 주문 실행기: 반드시 guard 통과 후에만 broker 주문 호출 (매수/매도 한글 주석 필수)
├─ strategy/
│   ├─ base.py             # 전략 공통 인터페이스(감시→조건→주문 흐름)
│   └─ stoploss.py         # 전략 ①: 스탑로스(손절)
└─ logs/                   # 일별 로그(개발 중 '계기판') — git 제외
```

**핵심 규칙:** `strategy`와 `telegram_bot`은 **절대 `broker.order`를 직접 부르지 않는다.** 반드시 `core/order.py` → `safety/guard.py` 검문을 거친다. "주문으로 가는 길은 오직 검문소 하나"로 강제하는 것이 이 구조의 존재 이유.

## 리스크 대응 (AUDIENCES.md의 공통 위험 → 개발 단계 방어책)

경쟁 서비스 사용자들이 가장 많이 데인 지점은 **"시뮬은 좋았는데 실전은 급손실"** 이었다. 톡트레이더도 예외가 아니므로 코드에 미리 심는다.

- **실전 괴리 · 슬리피지** *(Composer/젠포트 공통 불만)* → 시장가 주문 시 **예상가 vs 실제 체결가 차이를 항상 로그**로 남기고, **저유동 종목은 화이트리스트/한도로 제한**. 모의에서 체결 지연·미체결도 관찰.
- **급손실** *(Composer "이틀 만에 -30%")* → **모의 검증 게이트를 절대 건너뛰지 않기** + 일일 손실/주문 한도를 guard에 내장. 실전 전환은 소액부터.
- **API키·전략 유출** *(젠포트 해킹·전략 탈취 교훈)* → `.env`는 커밋 금지·`.gitignore` 등록·**커밋 전 시크릿 스캔**. 전략 코드는 내 PC에만.
- **기대치 관리** *(핀트 1★=단기 수익 실망)* → v1 성공지표를 **수익률이 아니라 "정확성·안전성·무사고 완주"** 로 둔다(MISSION과 정렬). "대박"이 목표가 아님을 스스로 명확히.
