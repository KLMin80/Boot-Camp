# 🔗 제휴(수익화) 신청 가이드 — 남김없이

앱 코드·고지 UI는 **이미 준비 완료**예요. 남은 건 **당신이 제휴에 신청**하고, 승인되면 **`.env`에 링크 한 줄**만 넣는 것.

---

## 지금 상태 (코드 쪽은 끝)

- ✅ 멀티마켓 링크는 `server.js`의 `MARKETS` 한 곳에서 생성 (쿠팡·컬리·이마트몰·네이버)
- ✅ **`.env`로 교체 가능** — 승인 후 `MARKET_URL_<KEY>` 템플릿만 넣으면 코드 수정·재빌드 없이 딥링크로 바뀜 (검색어 자리 = `{q}`)
- ✅ **제휴 고지 UI 탑재** — 구매 시트에 ‘제휴’ 배지 + "구매 시 수수료를 받을 수 있고, 가격은 동일" 문구 (표시·광고의 공정화에 관한 법률상 필수)

---

## 1) 쿠팡파트너스 — **가장 현실적인 수익원 (개인 가능)**

**신청:** [partners.coupang.com](https://partners.coupang.com) → 가입 → 본인인증
- 필요: 실명 본인인증, **정산 계좌**, (수익 발생 시)세금용 정보, **채널(사이트/앱) URL 등록**
- 채널 URL에 배포된 앱 넣기: `https://nowaste-app-seven.vercel.app`
- 심사 후 승인 (트래픽·콘텐츠가 어느 정도 필요할 수 있음. 지금 앱이 배포돼 있으니 등록 가능)

**승인 후 적용 (코드 안 건드림):**
1. 파트너스에서 **검색 딥링크**(또는 다이나믹 링크) 생성 — 트래킹 ID(subId)가 붙은 URL을 받음
2. Vercel 환경변수(또는 `.env`)에 추가:
   ```
   MARKET_URL_COUPANG=https://link.coupang.com/re/…?q={q}&subId=nowaste
   ```
   (파트너스가 준 실제 형식에 맞추고, 검색어 자리에 `{q}`만 남기면 됨)
3. 재배포 → 끝. 앱의 쿠팡 버튼이 제휴 링크로 바뀜.

> ⚠️ 쿠팡파트너스 정책: 카톡 등 **일부 채널 공유 제한**, 고지 문구 필수(이미 앱에 있음), 자기 클릭 금지 등 약관 준수.

---

## 2) 네이버쇼핑 — **개인 제휴는 확인 필요 (솔직히 애매)**

네이버는 쿠팡파트너스처럼 "개인이 쇼핑 검색 링크로 커미션"을 받는 정식 경로가 **명확하지 않아요.**
- **네이버 애드포스트**: 블로그·미디어 광고 수익용 (앱 검색링크와 성격 다름)
- **네이버 쇼핑파트너센터**: 판매자(입점)용
- → 개인 앱의 쇼핑 검색 링크 제휴가 되는지 **네이버 정책을 직접 확인**하고, 되면:
  ```
  MARKET_URL_NAVER=https://…?query={q}&NaPm=…(트래킹)
  ```
- 안 되면 네이버는 **‘참고가·비교용’**으로만 두고 (지금처럼), 제휴 배지만 내리면 됨(`server.js` MARKETS의 `naver` affiliate를 false로).

---

## 3) 마켓컬리 · 이마트몰 — 개인 제휴 없음

개인 대상 공식 제휴 프로그램이 없어요. 지금처럼 **검색 링크(비교용)**로만 제공 (affiliate=false, 배지 없음). 나중에 제휴 파트너십이 생기면 동일하게 `MARKET_URL_KURLY` / `MARKET_URL_SSG`로 교체.

---

## 현실적인 결론

- **당장 수익화 = 쿠팡파트너스 하나에 집중.** 신청→승인→`.env` 한 줄.
- 네이버는 정책 확인 후 되면 추가, 안 되면 비교용 유지.
- 손익분기(COST.md): 가구당 월 운영비 ~₩233 → **쿠팡 제휴로 월 1만 원대만 전환돼도 상쇄.**

## 내가(개발자가) 해둔 것 vs 당신이 할 것

| | 상태 |
|---|---|
| 링크 교체 구조(`.env`) | ✅ 완료 |
| 제휴 고지 UI(법적) | ✅ 완료 |
| 쿠팡파트너스 신청 | ⬜ **당신** (본인인증·정산정보) |
| 승인 후 `.env`에 링크 | ⬜ 알려주시면 제가 넣고 재배포 |
